- **A secret submitted via `env` no longer sits plaintext in the SQLite file forever.** Job rows are kept indefinitely by default (`JOBD_JOB_RETENTION_DAYS=0`), so every `HF_TOKEN`/`WANDB_API_KEY` ever submitted lived in `jobs.env_json` — and in every backup — for the life of the deployment, readable by anyone with access to the DB file. The sweeper now masks a terminal job's env **values** (keys kept, same `{"KEY": "***"}` shape every read surface already returns) once it has been finished for `JOBD_ENV_SCRUB_HOURS` (default 1; negative disables). Nothing observable changes — reads were always masked and only the `/next-job` claim ever consumes real values, which no terminal row can reach again. The one terminal→QUEUED restore path (a cascade-cancelled child a parent resurrect can un-cancel) is excluded via its `parent_failed:` warning stamp — now a shared constant with the un-cascade's matcher so the two predicates cannot drift — and a test pins that a restorable child keeps its real env no matter how old.
