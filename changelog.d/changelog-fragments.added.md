- **Per-PR changelog fragments (`changelog.d/`) replace editing `[Unreleased]` directly.** Five parallel PRs in the v0.5.26 merge train conflicted on the same `CHANGELOG.md` lines at every single merge — the shared-insertion-point design guaranteed it. Each PR now adds its own `changelog.d/<slug>.<category>.md` file (no shared lines, no conflicts); `scripts/roll-changelog.py <version>` assembles them into the released section at release time and deletes the consumed fragments. The scheme is enforced by real-execution tests: the roll script runs against fixture repos (happy path, unknown category, malformed bullet, stray `[Unreleased]` content, duplicate version — all refused with the changelog untouched), and `[Unreleased]` staying empty is a standing assertion, so an entry pasted the old way fails CI with a pointer to the new home.
